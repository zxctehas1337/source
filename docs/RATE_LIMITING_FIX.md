# Исправление Rate Limiting

## Проблема
При использовании премиум чата возникала ошибка 429 "Слишком много запросов" даже при небольшом количестве запросов.

## Причина
Слишком строгий `apiLimiter` - всего 30 запросов в минуту для всех API эндпоинтов.

## Решение
1. Увеличен лимит `apiLimiter` с 30 до 100 запросов в минуту
2. Добавлен новый `readLimiter` для GET запросов - 200 запросов в минуту
3. Применен `readLimiter` к следующим эндпоинтам:
   - `GET /api/users/:id` - получение информации о пользователе
   - `GET /api/users` - получение всех пользователей
   - `GET /api/news` - получение новостей
   - `GET /api/news/:newsId/comments` - получение комментариев
   - `GET /api/premium-chat` - получение сообщений чата
   - `GET /api/analytics/stats` - получение статистики

## Текущие лимиты
- **generalLimiter**: 100 запросов за 15 минут (применяется ко всем запросам)
- **authLimiter**: 5 попыток входа за 15 минут
- **apiLimiter**: 100 запросов в минуту (для POST/PUT/PATCH/DELETE)
- **readLimiter**: 200 запросов в минуту (для GET запросов)

## Результат
Теперь пользователи могут свободно использовать премиум чат и другие функции без ошибок rate limiting.
